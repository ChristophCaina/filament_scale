## Vision

The Filament Scale is more than just a weighing device — it is a **universal bridge between all major filament management ecosystems**.

While most solutions are locked to a single ecosystem, this project connects them all:

| Ecosystem | Protocol | Support |
| :--- | :--- | :--- |
| **Home Assistant** | ESPHome Native API | ✅ Full integration |
| **Spoolman** | REST API (PATCH) | ✅ Weight sync |
| **FilaMan** | Spoolman-compatible API | ✅ Weight sync, Tag-Creation / Update |
| **Mifare / NTAG Tags** | ISO 14443 via PN532 | ✅ v1 |
| **OpenPrintTag (Prusa)** | ISO 15693 via PN5180 | 🔲 v2 |

> 💡 FilaMan currently uses a PN532 reader and cannot read OpenPrintTag (ISO 15693) tags. This scale — with a PN5180 — can act as a bridge: reading OpenPrintTag spools and syncing their data to FilaMan or Spoolman.

## Architecture / Package Structure

```
user-config.yaml             ← Local device file (user-owned, never overwritten)
│
├── substitutions:           ← Hardware-specific values (pins, IPs)
│   └── (empty = HA entity has priority)
│   └── (set = substitution overrides on boot)
│
├── api: encryption: key:    ← Generated by ESPHome on Take Control
├── wifi:                    ← User's WiFi credentials
├── ota:                     ← ESPHome OTA platform
│
└── packages:
    └── filament_scale:      ← github://...filament-scale.yaml@main
                                (contains all functionality)
```

```
filament-scale.yaml          ← Core package on GitHub (all functionality)
│
└── packages:
    ├── base.yaml            ← Status LED, Buzzer, Beep scripts
    ├── scale.yaml           ← HX711: calibration, tare, weight, filament %
    ├── display.yaml         ← ST7920: menu rendering, pagination
    ├── controls.yaml        ← Rotary encoder: navigation input
    ├── sync.yaml            ← Unified Spoolman/FilaMan sync
    ├── stepper.yaml         ← Stepper motor control
    ├── winder.yaml          ← Winder add-on detection
    └── nfc.yaml             ← PN532 NFC tag reading/writing
```

```
factory.yaml                 ← Web Installer only (initial flash)
│                               Contains: improv_serial, captive_portal,
│                               api (no key), ota, dashboard_import
└── packages:
    └── filament_scale:      ← same filament-scale.yaml
```

## Key Design Decisions

### Two-tier architecture: factory.yaml + user-config.yaml

The most important architectural decision: **credentials and API keys never live on GitHub**.

- `factory.yaml` — flashed once via Web Installer, contains no credentials
- `user-config.yaml` — generated by ESPHome "Take Control", contains API key + WiFi
- OTA updates via ESPHome Dashboard recompile `user-config.yaml` → **API key survives updates**
- OTA updates via Home Assistant flash `factory.yaml` binary → for users without ESPHome

### Normal User vs. Power User

All configurable values (calibration factor, Spoolman/FilaMan IP) follow a priority system:

```
Substitution empty  →  HA entity has priority  (Normal User)
Substitution set    →  Substitution overrides on boot  (Power User)
```

Normal Users configure everything via Home Assistant UI. Power Users set substitutions in their local YAML — these persist across OTA updates.

### Why ESPHome?

- No custom firmware needed — everything is YAML
- Native Home Assistant integration
- OTA updates without cloud dependency
- Web Installer for first-time setup (no drivers, no software)
- Active community and long-term support

### Why ST7920 + Rotary Encoder?

Simple, reliable, low-power. Wide availability as Ender 3/5 spare part — many makers already have one. No touch calibration complexity near metal parts.

### Why HX711?

De facto standard for DIY load cell interfaces. 24-bit resolution, built-in amplification, excellent ESPHome support. Calibration is done via Home Assistant number entity — no reflash needed.

### Winder as Optional Add-on

Auto-detected via GPIO05 (pulled high, grounded when add-on connected). Base device works as a standalone scale without winder attached. No separate firmware variant needed.

## Boot Flow

```
Boot
 │
 ├─► Connect WiFi
 │    ├─► Connected → register with Home Assistant
 │    └─► Failed → start Fallback AP + Captive Portal
 │
 ├─► on_boot: apply substitutions (if set)
 │    ├─► scale_calibration_factor → calibration_factor entity
 │    ├─► spoolman_ip → text_spoolman_ip entity
 │    └─► filaman_ip → text_filaman_ip entity
 │
 ├─► script_detect_backend
 │    ├─► FilaMan IP set → use FilaMan API
 │    ├─► Spoolman IP set → use Spoolman API
 │    └─► Neither set → sync disabled
 │
 ├─► Check Add-on GPIO (GPIO05)
 │    ├─► Winder detected → enable winder menu items
 │    └─► No winder → hide winder menu items
 │
 └─► Main Loop
      ├─► Read weight (scale.yaml) → update HA sensors
      ├─► NFC tag detected? → read ID → set spool_id → sync
      ├─► Encoder input → navigate menu
      └─► Winder active? → read potentiometer → set stepper speed
```

## Vision

Die Filament-Waage ist mehr als ein Wiegegerät — sie ist eine **universelle Brücke zwischen allen wichtigen Filament-Management-Ökosystemen**.

Während die meisten Lösungen auf ein einzelnes Ökosystem beschränkt sind, verbindet dieses Projekt alle miteinander:

| Ökosystem | Protokoll | Unterstützung |
| :--- | :--- | :--- |
| **Home Assistant** | ESPHome Native API | ✅ Vollständige Integration |
| **Spoolman** | REST API (PATCH) | ✅ Gewichtssync |
| **FilaMan** | Spoolman-kompatibler API | ✅ Gewichtssync / Tag-Erstellung, Tag-Update |
| **Mifare / NTAG Tags** | ISO 14443 via PN532 | ✅ v1 |
| **OpenPrintTag (Prusa)** | ISO 15693 via PN5180 | 🔲 v2 |

> 💡 FilaMan nutzt aktuell einen PN532-Reader und kann keine OpenPrintTag-Tags (ISO 15693) lesen. Diese Waage kann — mit einem PN5180 — als Brücke fungieren: OpenPrintTag-Spulen lesen und die Daten an FilaMan oder Spoolman übertragen.

## Wichtige Designentscheidungen

### Zweistufige Architektur: factory.yaml + user-config.yaml

Die wichtigste Architekturentscheidung: **Zugangsdaten und API-Keys leben nie auf GitHub**.

- `factory.yaml` — einmalig per Web Installer geflasht, enthält keine Zugangsdaten
- `user-config.yaml` — von ESPHome "Take Control" generiert, enthält API-Key + WLAN
- OTA-Updates via ESPHome Dashboard kompilieren `user-config.yaml` neu → **API-Key überlebt Updates**
- OTA-Updates via Home Assistant flashen `factory.yaml` Binary → für Nutzer ohne ESPHome

### Normal User vs. Power User

Alle konfigurierbaren Werte (Kalibrierungsfaktor, Spoolman/FilaMan-IP) folgen einem Prioritätssystem:

```
Substitution leer  →  HA-Entity hat Priorität  (Normal User)
Substitution gesetzt  →  Substitution überschreibt beim Boot  (Power User)
```

Normal User konfigurieren alles über die Home Assistant UI. Power User setzen Substitutions in ihrer lokalen YAML — diese bleiben über OTA-Updates hinweg erhalten.

### Warum ESPHome?

- Keine eigene Firmware nötig — alles ist YAML
- Native Home-Assistant-Integration
- OTA-Updates ohne Cloud-Abhängigkeit
- Web Installer für die Ersteinrichtung (keine Treiber, keine Software)
- Aktive Community und langfristiger Support

### Warum ST7920 + Drehgeber?

Einfach, zuverlässig, stromsparend. Weit verbreitet als Ender-3/5-Ersatzteil — viele Maker haben bereits eines. Keine Touch-Kalibrierungskomplexität in der Nähe von Metallteilen.

### Warum HX711?

De-facto-Standard für DIY-Wägezelleninterfaces. 24-Bit-Auflösung, eingebaute Verstärkung, exzellente ESPHome-Unterstützung. Die Kalibrierung erfolgt über eine Home-Assistant-Number-Entity — kein erneutes Flashen erforderlich.

### Wickler als optionales Add-on

Automatische Erkennung via GPIO05 (High-gezogen, bei angestecktem Add-on auf GND). Das Basisgerät funktioniert als eigenständige Waage ohne Wickler. Keine separate Firmware-Variante nötig.

## Startablauf

```
Start
 │
 ├─► WLAN verbinden
 │    ├─► Verbunden → bei Home Assistant registrieren
 │    └─► Fehlgeschlagen → Fallback-AP + Captive Portal starten
 │
 ├─► on_boot: Substitutions anwenden (falls gesetzt)
 │    ├─► scale_calibration_factor → calibration_factor Entity
 │    ├─► spoolman_ip → text_spoolman_ip Entity
 │    └─► filaman_ip → text_filaman_ip Entity
 │
 ├─► script_detect_backend
 │    ├─► FilaMan-IP gesetzt → FilaMan-API verwenden
 │    ├─► Spoolman-IP gesetzt → Spoolman-API verwenden
 │    └─► Keine gesetzt → Sync deaktiviert
 │
 ├─► Add-on-GPIO prüfen (GPIO05)
 │    ├─► Wickler erkannt → Wickler-Menüpunkte aktivieren
 │    └─► Kein Wickler → Wickler-Menüpunkte ausblenden
 │
 └─► Hauptschleife
      ├─► Gewicht lesen (scale.yaml) → HA-Sensoren aktualisieren
      ├─► NFC-Tag erkannt? → ID lesen → Spool-ID setzen → Sync
      ├─► Drehgeber-Eingabe → Menü navigieren
      └─► Wickler aktiv? → Potentiometer lesen → Schrittmotorgeschwindigkeit setzen
```
